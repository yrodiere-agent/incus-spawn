# isx.run hero cast — recording script

Target: **≤ 90 seconds**, one take (edited), real terminal, no slides. The cast must prove the loop: *branch → agent works on its own machine → review as a commit → destroy* — and land the credential-isolation punch in passing.

## The story beats (what a viewer should be able to retell)

1. I created a machine for the agent in ~2 seconds.
2. The machine contains **no secrets** — yet Claude works anyway.
3. The agent fixed a bug and committed it, alone on its own machine.
4. I reviewed the commit from my host with plain `git`, took it, and threw the machine away.

## Pre-flight checklist (do all of this BEFORE recording)

- [ ] Terminal: **100×28**, dark theme, font large enough to read at 720px wide (test: `asciinema play` in a half-width window). Plain prompt, no powerline glyphs that might not render in the web player.
- [ ] `isx doctor` — everything green.
- [ ] Proxy running; Claude auth working (do one warm-up run so nothing prompts).
- [ ] Template `tpl-demo` built (see appendix A) — includes the demo repo cloned + primed, `claude`, `workdir` set.
- [ ] Demo repo (appendix B) pushed to GitHub with the **planted bug on `main`** and a failing test → done: https://github.com/Sanne/isx-demo
- [ ] Host checkout at `~/sources/isx-demo` (already covered by the existing `host-paths: [~/sources]`, so the auto-remote appears on branch) — checked out at `main`, clean.
- [ ] No instance named `fix-bug` (destroy leftovers from rehearsals).
- [ ] Warm-up: run the whole scenario once end-to-end. Claude should fix the planted bug in a single attempt; if it wanders, simplify the bug or sharpen the task prompt.
- [ ] Record with: `asciinema rec --idle-time-limit 1.5 --cols 100 --rows 28 hero.cast`

## Shot list

Typed `#` comments are the narration — keep them short, type them confidently (idle-limit trims your pauses, not your typos).

### Beat 1 — a machine in seconds (~0:00–0:12)

```shell
# give the agent its own machine:
isx branch --from tpl-demo fix-bug
```

*(Output shows the CoW clone, the auto-added git remote, and then drops straight into the machine — `isx branch` auto-shells. That speed IS the demo, don't talk over it.)*

### Beat 2 — no secrets inside, yet everything authenticates (~0:12–0:35)

```shell
# no secrets in this machine:
env | grep TOKEN
```

*(Prints `GH_TOKEN=gho_placeholder` — one clean line. Don't broaden the pattern: wider probes surface host details like the Vertex project id.)*

```shell
printenv ANTHROPIC_API_KEY || echo 'anthropic: nothing at all'
```

*(Prints `anthropic: nothing at all` — in Vertex mode the container holds no Anthropic credential in any form, not even a placeholder.)*

```shell
# yet everything authenticates. watch:
gh api user --jq .login
```

*(Prints the bot identity — `Sanne-ai-bot` — proving the placeholder authenticates AND that the agent acts under its own identity, not yours.)*

```shell
claude
```

### Beat 3 — the agent works (~0:25–0:50 after editing)

Task prompt to type into Claude:

> The test suite fails. Find the bug, fix it, run the tests, and commit the fix.

*(In reality this takes 1–3 minutes. Let it run; we cut it down in editing — keep the first tool call, one glimpse of the diff, and the final "committed" message. See "Editing" below.)*

When Claude finishes: exit Claude, then `exit` the shell.

### Beat 4 — review from the host, then destroy (~0:50–1:15)

```shell
cd ~/sources/isx-demo

# Review the agent's work — an immutable commit, over plain git:
git fetch fix-bug
git diff main..fix-bug/main
```

*(The diff is the payoff frame: a small, obviously-correct fix. Pause 2s on it.)*

```shell
# Take it. Then throw the machine away:
git cherry-pick fix-bug/main
isx destroy fix-bug
```

*(Destroy output shows the remote being cleaned up too.)*

### Beat 5 — closing line (~1:10–1:20)

```shell
# every action traceable to the agent — nothing arrives uninvited. credentials never left the host.  ∎
```

*(One line, typed without pressing Enter so no fresh prompt appears; the cast ends frozen on it. Claude is exited with a double Ctrl+D — quieter on camera than `/exit`, which flashes the slash-command menu.)*

## Automated recording (how these casts were actually made)

All four casts are generated by `record.py`, which drives a real PTY against a live isx installation — the terminal output is genuine, only the keystrokes are scripted (with human-ish jitter). One scenario JSON per cast:

```shell
python3 record.py scenario-hero.json hero.cast \
  --cmd "cd \$HOME/sources/isx-demo; PATH=\$HOME/.local/bin:\$PATH PS1='➜ ' GIT_PAGER=cat BASH_SILENCE_DEPRECATION_WARNING=1 exec bash --norc"
```

Post-processing with `compress.py` (idle-gap capping, fast-forward cuts, speed scaling) produces the embeddable `-web.cast` files, e.g.:

```shell
python3 compress.py hero.cast hero-web.cast --max-gap 0.8 --scale 1.22 --cut 75:91 --cut 96:120
```

(Cut ranges are in raw-cast seconds and shift with every retake — find fresh anchors by grepping the cast for marker text before cutting.)

Retake protocol: `isx destroy` leftover instances, `git -C ~/sources/isx-demo reset --hard origin/main` (restores the planted bug and drops the on-camera cherry-pick), re-run the scenario.

Gotchas learned the hard way:
- **sudo logs the credential hunt into the journal**, so the grep finds its own audit trail — search for `'sk-ant-ap[i]'` so the pattern can't match its own literal.
- **Containers remember rehearsals** (`.bash_history`, journal) — the creds cast needs a fresh branch per take.
- **Broad env patterns surface host details** (e.g. the Vertex project id) — probe `env | grep TOKEN`, not `ANTHROPIC|KEY`.
- Nothing on the *host* side of a recording may touch credentials: no `cat config.yaml`, no `isx proxy logs`.

## Editing the cast

The `.cast` file is newline-delimited JSON — editable. Two passes:

1. `--idle-time-limit 1.5` at record time already compresses thinking pauses.
2. Cut Beat 3's middle with [`asciinema-edit`](https://github.com/cirocosta/asciinema-edit): `asciinema-edit cut --start=<t1> --end=<t2> hero.cast` — keep ~15–20s of agent activity total. A hard cut mid-scroll looks fine in terminal recordings; nobody expects continuity in scrolling output.
3. Verify total length ≤ 90s: last timestamp in the file, or just `asciinema play hero.cast`.

Drop the final `hero.cast` into `site/demo/` — embedding it in the homepage (asciinema-player, self-hosted, poster frame on the diff payoff) is a separate small task I can do once the file exists.

## Appendix A — demo template

```yaml
# ~/.config/incus-spawn/images/tpl-demo.yaml
name: tpl-demo
description: Demo template for the isx.run hero cast
parent: tpl-java
repos:
  - url: https://github.com/Sanne/isx-demo.git
    path: ~/demo-app
    prime: mvn -B -q test-compile
workdir: ~/demo-app
tools:
  - claude
default-action: claude
```

The `prime` step pre-fetches all Maven dependencies into the template, so the agent's `mvn test` runs in seconds on camera instead of downloading the internet.

## Appendix B — demo repo (`isx-demo`)

A deliberately tiny Maven project with one planted, single-line bug. Java/Maven is the right choice for the audience, and the warm `mvn test` also quietly demos template priming.

```
isx-demo/
  pom.xml                  # minimal: junit-jupiter, surefire; java 17
  src/main/java/demo/PriceCalculator.java
  src/test/java/demo/PriceCalculatorTest.java
```

`PriceCalculator.java` — the planted bug is `+` instead of `-`:

```java
package demo;

public class PriceCalculator {

    /** Applies a percentage discount to a price. */
    public static double applyDiscount(double price, double discountPercent) {
        return price + price * (discountPercent / 100.0);  // BUG: should subtract
    }
}
```

`PriceCalculatorTest.java`:

```java
package demo;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

class PriceCalculatorTest {

    @Test
    void tenPercentOffOneHundred() {
        assertEquals(90.0, PriceCalculator.applyDiscount(100.0, 10.0), 0.001);
    }
}
```

Why this bug: it's discoverable from the failing assertion alone (expected 90, got 110), fixable in one line, and the resulting `git diff` in Beat 4 is readable at a glance in a terminal — the whole review fits in one frame.

## Appendix C — section-cast staging

**branching.cast (the database fork)** needs a `workbench` instance with seeded postgres. Prep (off camera, once):

```shell
isx branch --from tpl-demo workbench
# inside:
sudo dnf install -y postgresql-server
sudo /usr/bin/postgresql-setup --initdb
sudo systemctl enable --now postgresql
sudo -u postgres createuser -s agentuser && sudo -u postgres createdb agentuser
psql -qc 'create table orders as select generate_series(1,100000) as id, md5(random()::text) as sku;'
```

The scenario branches `exp-a`/`exp-b` from `workbench`; destroy those after each take, keep `workbench` for retakes. The 3s pause after each branch lets postgres finish WAL recovery from the fork before `psql` runs.

**tui.cast**: the TUI must show a clean board, so run it under a staged home: `isx -Duser.home=/tmp/isx-demo-home` (the native binary ignores `$HOME` but honors java-style `-D` flags). Stage `/tmp/isx-demo-home/.config/incus-spawn/` with the real `config.yaml` minus its `searchPaths`, `ca.crt`/`ca.key`, and only `images/tpl-demo.yaml`. Built templates and instances still come from the daemon regardless of config — pre-stage realistic instances (`fix-flaky-test` running, `triage-issue-8812` stopped) and destroy leftover build debris first. The scenario drives the TUI blind (arrows, F4, modal typing) and waits on Claude's `bypass` status-bar text as the readiness marker; the clip deliberately ends frozen on Claude open — no exit choreography. Destroy `review-pr-512` after each take.

**realmachine.cast** prep: in the target instance, pre-warm dnf metadata (`sudo dnf makecache`) and remove the demo package (`sudo dnf remove -y btop`; `podman rmi quay.io/podman/hello`) so the on-camera install is quick and honest.

**creds.cast**: always record on a *fresh* branch — containers remember rehearsals (`.bash_history`, journal), and the credential grep must find nothing.

## Rehearsal notes

- Do at least two full rehearsals; record the third.
- If Claude asks a clarifying question instead of acting, prepend "Don't ask questions —" to the task prompt, or tighten the repo (fewer files = less to explore).
- Keep your typing rhythm steady; resist narrating verbally (the cast is silent — the `#` comments are the voice-over).
- If a take goes sideways: `isx destroy fix-bug`, `git -C ~/sources/isx-demo reset --hard origin/main` (drops the cherry-pick), and go again. Cheap takes are the whole point of the product — if that ever stops being funny, take a break.
